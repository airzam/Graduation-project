# 树莓派 5 网络启动配置

通过 Dnsmasq 搭建 PXE/TFTP 服务器，实现不拔 SD 卡通过网络更新 EFI 程序。

## 整体架构

```
电脑 (VMware Ubuntu)                      路由器                       树莓派 5
192.168.186.135                           192.168.186.254               有线网口
     │                                          │                          │
     │ ←────── 网线桥接/直连 ──────────────→    │ ←────── 网线 ──────────→ │
     │                                          │                          │
     ├── Dnsmasq (DHCP Proxy + TFTP)           │     正常 DHCP 分配 IP     │
     │    监听 UDP 67                           │                          │
     └── TFTP Server (/srv/tftp/)               │                          │
                                               │                          │
                                               │ ←── PXE 请求广播 ─────────┤
                                               │ ──→ TFTP 路径回应 ───────→│
                                               │                          │
                                               │ ←── 下载 MyGuiFrame.efi ─┤
                                               │ ──→ 传输完成，启动 ──────→│
```

**前提条件**：
- 树莓派用**网线连接**到路由器（WiFi 无法 PXE 启动）
- 电脑与树莓派在同一网络

---

## 第一步：电脑端安装 Dnsmasq

```bash
sudo apt update
sudo apt install dnsmasq
```

## 第二步：创建 TFTP 目录

```bash
sudo mkdir -p /srv/tftp
sudo chmod 777 /srv/tftp
```

## 第三步：复制 EFI 文件

```bash
# 复制编译好的 EFI 文件到 TFTP 目录
sudo cp ~/Graduation-project/rpi5-uefi/edk2/Build/MyAppPkg/DEBUG_GCC5/AARCH64/MyGuiFrame.efi /srv/tftp/MyGuiFrame.efi

# 每次重新编译后需要执行此步更新
```

## 第四步：配置 Dnsmasq

```bash
sudo nano /etc/dnsmasq.d/pxe-boot
```

写入以下内容：

```conf
# 监听接口（连接路由器的网口）
interface=ens33

# 关闭 DNS 功能（只做 DHCP Proxy）
port=0

# DHCP Proxy 模式：不分配 IP，只响应 PXE 请求
# 192.168.186.0 是你的网络号，请根据实际情况修改
dhcp-range=192.168.186.0,proxy

# PXE 启动文件名
dhcp-boot=MyGuiFrame.efi

# TFTP 服务器地址（电脑 IP）
dhcp-option=option:server-name,"192.168.186.135"

# 日志（调试时开启）
log-dhcp
log-queries
```

**注意**：将 `192.168.186.135` 改为你的电脑实际 IP（可用 `hostname -I` 查看）。

## 第五步：修改 Dnsmasq 主配置

```bash
sudo nano /etc/dnsmasq.conf
```

确保以下内容存在：

```conf
strict-order
interface=ens33
```

## 第六步：重启 Dnsmasq 服务

```bash
sudo systemctl restart dnsmasq
sudo systemctl enable dnsmasq
```

## 第七步：验证服务状态

```bash
# 检查 Dnsmasq 是否运行
sudo systemctl status dnsmasq

# 检查 TFTP 目录
ls -la /srv/tftp/

# 查看实时日志
sudo tail -f /var/log/syslog | grep dnsmasq
```

## 第八步：树莓派端配置

### 8.1 启用网络启动

1. 插入 SD 卡，启动树莓派 5
2. 按 **ESC** 或 **DEL** 进入 UEFI Setup
3. 进入 **Device Manager** → **Network Stack** 或 **PXE Configuration**
4. 启用 **Network Boot** 或 **UEFI PXE**

### 8.2 设置启动顺序

1. 进入 **Boot Manager**
2. 找到 **UEFI PXE** 或 **Network Boot** 选项
3. 按 **Enter** 将其移动到启动顺序 **第一位**

### 8.3 保存并重启

1. 按 **F10** 保存设置
2. 确认重启
3. 屏幕显示 "Start PXE over IPv4" 或类似文字

---

## 第九步：测试网络启动

1. 确保树莓派网线已连接路由器
2. 确保电脑端 Dnsmasq 已启动
3. 重启树莓派

**成功情况**：
- 屏幕上显示 "PXE booting..." → "Downloading NBP file..." → 加载 EFI 程序

**失败情况**：
- 继续从 SD 卡启动（说明 PXE 未生效）
- 显示错误信息（如 "No DHCP response"）

## 每次更新代码的流程

```bash
# 1. 重新编译 EFI
cd ~/Graduation-project/rpi5-uefi/edk2
export GCC5_AARCH64_PREFIX=aarch64-linux-gnu-
source edksetup.sh
build -a AARCH64 -t GCC5 -b DEBUG -p MyAppPkg/MyAppPkg.dsc

# 2. 复制到 TFTP 目录
sudo cp ~/Graduation-project/rpi5-uefi/edk2/Build/MyAppPkg/DEBUG_GCC5/AARCH64/MyGuiFrame.efi /srv/tftp/MyGuiFrame.efi

# 3. 树莓派重启即可自动从网络加载新程序
```

## 常见问题

### Q1: Dnsmasq 启动失败

```bash
# 查看错误日志
sudo journalctl -u dnsmasq -n 20

# 检查接口是否存在
ip addr show ens33
```

### Q2: 树莓派没有进入 PXE 启动

- 确认启动顺序中 PXE 在 SD 卡之前
- 确认网线连接正常（路由器指示灯亮）
- 有些 UEFI 固件需要先移除 SD 卡才能网络启动

### Q3: 如何恢复到 SD 卡启动？

在 UEFI Setup → Boot Manager 中，将 **SD Card** 移到第一位，或进入 Boot Override 选择 SD 卡启动。
