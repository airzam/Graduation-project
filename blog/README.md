# 博客编写规范

## 文件命名

```
序号-标题.md
例：01-windows虚拟机安装ubuntu和claude-code.md
```

## 文章结构

### 标题
- 一级标题：`# 标题`
- 放在文件最开头

### 开头
- 简短引言（1-2句话说明本文要讲什么）
- 不需要"前言"、"摘要"等小标题

### 内容
- 用层级标题组织内容（如 `## 一、`、`## 二、` 或 `## 第一步`、`## 第二步`）
- 只写关键步骤和必要说明
- 删除冗余的解释性文字、客套话

### 代码
- 使用代码块，标明语言
- 命令用 `bash`，代码用对应语言

### 结尾
- 不需要"总结"、"结语"等
- 如需延伸阅读，放在最后

## 风格要求

- 简洁直接，不废话
- 多用列表、表格
- 关键步骤和代码要完整准确
- 删除所有"例如："、"一般来说"、"通常情况下"等冗余表达
- 删除打招呼、感谢阅读等客套话

## 图片插入

```markdown
![图片说明](./images/图片名.png)
```

图片放在 `images` 文件夹中。

## 示例

```markdown
# 文章标题

简短引言。

## 第一步

操作说明。

```bash
命令代码
```

## 第二步

操作说明。

```bash
命令代码
```

## 延伸

- [参考链接](url)
```

## AI 编写提示词模板

如果让 AI 帮你写博客，可以用以下提示：

```
帮我写一篇博客，主题是：xxx

要求：
- 参考上面的博客规范
- 只写关键步骤和代码
- 简洁直接，不废话
- 文件保存到 blog/序号-标题.md
```
