# 树莓派 5 串口通信配置

通过串口实现电脑与树莓派的通信，可用于调试和传输文件。

## 整体架构

```
电脑 (USB转TTL)                      树莓派 5
   |                                      |
   |---- TX/RX/GND 串口线 ---->| GPIO 14/15
   |                                      |
   |-------- WiFi / USB -------->| 无线串口模块 (ESP8266透传)
```

**前提条件**：
- 电脑有 USB 接口连接串口模块（USB转TTL 或 无线串口模块）
- 树莓派 GPIO 串口可用（默认映射到调试串口）

## 方案一：串口回环测试

串口回环测试可以验证树莓派串口是否正常工作。

### 硬件连接（TTL 串口）

| ESP8266/TTL模块 | 树莓派 GPIO |
|----------------|-------------|
| TX | GPIO 15 (RXD) |
| RX | GPIO 14 (TXD) |
| GND | GND |

### 电脑端配置

1. 安装串口软件（如 PuTTY、SecureCRT、或 Linux 的 minicom）
2. 连接参数：
   - 波特率：115200
   - 数据位：8
   - 校验位：None
   - 停止位：1
   - 流控制：None

### 测试步骤

1. SD 卡插入树莓派，启动
2. 进入 UEFI Shell
3. 运行 `MyGuiFrame.efi`
4. 在串口终端看到输出 "Hello from UEFI!"
5. 按任意键程序退出

## 方案二：无线串口（ESP8266 透传）

使用 ESP8266 无线透传模块，实现 WiFi 串口传输。

### 硬件连接

| ESP8266 模块 | 树莓派 GPIO |
|-------------|-------------|
| TX | GPIO 15 (RXD) |
| RX | GPIO 14 (TXD) |
| GND | GND |
| 3.3V | 3.3V (谨慎供电，最好用独立电源) |

### ESP8266 配置

模块需要预先配置为透传模式（AT指令）：

```
AT+CIPMODE=1       # 透传模式
AT+CWJAP="SSID","PASSWORD"  # 连接 WiFi
AT+CIPSTART="TCP","192.168.1.100",8888  # 连接电脑 TCP 服务器
```

### 电脑端

使用 Python 脚本或串口软件连接：

```python
# Python 串口通信示例
import serial
import socket

# 方法1：直接串口连接
ser = serial.Serial('/dev/ttyUSB0', 115200)

# 方法2：通过 TCP 转串口（如 ESP8266 连接到电脑）
tcp = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
tcp.connect(('192.168.1.100', 8888))

# 发送数据
tcp.send(b'Hello RPi5!')
data = tcp.recv(1024)
print(data)
```

## SerialPkg 包结构

```
edk2/SerialPkg/                    # 串口通信包
├── SerialPkg.dec                  # 包声明
├── SerialPkg.dsc                  # 包构建配置
└── SerialRecv/                    # 串口回环测试应用
    ├── SerialRecv.inf             # 模块定义
    └── SerialRecv.c               # 主程序
```

### SerialRecv.c 核心代码

```c
#include <Uefi.h>
#include <Library/UefiLib.h>
#include <Library/UefiBootServicesTableLib.h>

EFI_STATUS EFIAPI UefiMain(
    IN EFI_HANDLE        ImageHandle,
    IN EFI_SYSTEM_TABLE *SystemTable
)
{
    EFI_INPUT_KEY Key;
    UINTN Count = 0;

    Print(L"===========================================\n");
    Print(L"  Serial Loopback Test\n");
    Print(L"===========================================\n");

    // Clear input buffer
    gST->ConIn->Reset(gST->ConIn, FALSE);

    while (1) {
        // Wait for key from serial
        EFI_STATUS Status = gST->ConIn->ReadKeyStroke(gST->ConIn, &Key);

        if (Status == EFI_SUCCESS) {
            Count++;

            // Display on screen
            if (Key.UnicodeChar != 0) {
                Print(L"%c", Key.UnicodeChar);
            } else if (Key.ScanCode == SCAN_ESC) {
                break;
            }

            // Echo back to PC via serial
            if (Key.UnicodeChar != 0) {
                CHAR16 Str[2] = { Key.UnicodeChar, L'\0' };
                gST->ConOut->OutputString(gST->ConOut, Str);
            }
        }

        gBS->Stall(1000);  // 1ms delay
    }

    Print(L"\nTotal: %d\n", Count);
    return EFI_SUCCESS;
}
```

---

## SerialRecv 工作原理

1. **读取串口输入**：`gST->ConIn->ReadKeyStroke()` 从串口读取字符
2. **屏幕显示**：通过 `Print()` 在屏幕上显示
3. **回传**：通过 `gST->ConOut->OutputString()` 发送回电脑
4. **循环**：持续读取直到按 ESC 键

## 常见问题

### Q1: 串口无输出

- 检查 TX/RX 是否接反
- 检查波特率是否正确（115200）
- 检查串口模块供电是否正常

### Q2: 乱码

- 波特率不匹配
- 数据位/校验位/停止位设置错误
- 尝试 8N1（8数据位，无校验，1停止位）

### Q3: ESP8266 连接失败

- 确认模块已配置为透传模式
- 确认电脑和 ESP8266 在同一网络
- 检查 TCP 端口是否被占用
