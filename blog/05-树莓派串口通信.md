# 树莓派 5 串口通信配置

通过串口实现电脑与树莓派的通信，可用于调试和传输文件。

## ⚠️ 重要说明

RPi5 UEFI 固件的串口**不是**连接在 GPIO 14/15 上！

| 接口 | 状态 | 说明 |
|------|------|------|
| **GPIO 14/15** | 🔴 不工作 | UART 复用功能被固件禁用（DMA 冲突） |
| **专用串口连接器** | 🟢 工作 | PL011 UART，3-pin 专用接口 |
| **GPIO** | 🟢 工作 | GIO/AON 针脚功能 |

**串口在 RPi5 板子上的独立 3-pin 插针**（非 40-pin GPIO），位置通常在 HDMI 接口附近或板子边缘，标注为 UART/TX/RX。

---

## 整体架构

```
电脑 (USB转TTL)                      树莓派 5
   |                                      |
   |---- TX/RX/GND 串口线 ---->| 专用串口连接器 (3-pin)
   |                                      |
   |-------- WiFi / USB -------->| 无线串口模块 (ESP8266透传)
```

**前提条件**：
- 电脑有 USB 接口连接串口模块（USB转TTL 或 无线串口模块）
- 树莓派专用串口已确认位置

## 方案一：串口回环测试

串口回环测试可以验证树莓派串口是否正常工作。

### 硬件连接（TTL 串口）

| USB转TTL 模块 | RPi5 专用串口 |
|--------------|-------------|
| TX | RX |
| RX | TX |
| GND | GND |

**注意**：不是连接 GPIO 14/15，而是板子上独立的 3-pin 串口插针

### 编译 SerialRecv.efi

```bash
cd ~/Graduation-project/rpi5-uefi/edk2
export GCC5_AARCH64_PREFIX=aarch64-linux-gnu-
source edksetup.sh
build -a AARCH64 -t GCC5 -b DEBUG -p SerialPkg/SerialPkg.dsc
```

### 复制到 SD 卡

```bash
# SD 卡挂载到 /mnt/sdcard
sudo cp ~/Graduation-project/rpi5-uefi/edk2/Build/SerialPkg/DEBUG_GCC5/AARCH64/SerialRecv.efi /mnt/sdcard/
```

### 电脑端配置

1. 安装串口软件（如 PuTTY、SecureCRT、或 Linux 的 minicom）
2. 连接参数：
   - 波特率：115200
   - 数据位：8
   - 校验位：None
   - 停止位：1
   - 流控制：None

### 测试步骤

1. SD 卡插入树莓派，启动
2. 进入 UEFI Shell
3. 运行 `SerialRecv.efi`
4. 屏幕显示 "Serial Loopback Test v2"，自动每 5 秒发送测试数据
5. 在串口终端应收到 "Hello from RPi5! (Auto test)"
6. 按 ESC 键退出

## 方案二：无线串口（ESP8266 透传）

使用 ESP8266 无线透传模块，实现 WiFi 串口传输。

### 硬件连接

| ESP8266 模块 | RPi5 专用串口 |
|-------------|-------------|
| TX | RX |
| RX | TX |
| GND | GND |
| 3.3V | 3.3V (谨慎供电，最好用独立电源) |

### ESP8266 配置

模块需要预先配置为透传模式（AT指令）：

```
AT+CIPMODE=1       # 透传模式
AT+CWJAP="SSID","PASSWORD"  # 连接 WiFi
AT+CIPSTART="TCP","192.168.1.100",8888  # 连接电脑 TCP 服务器
```

### 电脑端

使用 Python 脚本或串口软件连接：

```python
# Python 串口通信示例
import serial
import socket

# 方法1：直接串口连接
ser = serial.Serial('/dev/ttyUSB0', 115200)

# 方法2：通过 TCP 转串口（如 ESP8266 连接到电脑）
tcp = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
tcp.connect(('192.168.1.100', 8888))

# 发送数据
tcp.send(b'Hello RPi5!')
data = tcp.recv(1024)
print(data)
```

## SerialPkg 包结构

```
edk2/SerialPkg/                    # 串口通信包
├── SerialPkg.dec                  # 包声明
├── SerialPkg.dsc                  # 包构建配置
└── SerialRecv/                    # 串口回环测试应用
    ├── SerialRecv.inf             # 模块定义
    └── SerialRecv.c               # 主程序
```

### SerialRecv.c 核心功能

- 使用 `EFI_SERIAL_IO_PROTOCOL` 操作串口（不是 Console 协议）
- 枚举所有可用的 Serial I/O 端口
- 每 5 秒自动发送测试数据
- 回传 PC 发送的数据（回环测试）
- 支持 1024x600 分辨率显示

## 重要发现：GPIO 串口受限

在 `edk2-platforms/Platform/RaspberryPi/AcpiTables/Rhpx.asl` 中发现：

```c
// NTRAID#MSFT-7141401-2016/04/7jordanrh - disable UART muxing
// until a proper solution can be created for the dmap conflict
// GPIO 14 - UART TX
// GPIO 15 - UART RX
```

**原因**：GPIO 14/15 的 UART 复用功能因 DMA 冲突被固件禁用。

**解决方案**：使用 RPi5 板子上独立的 3-pin 专用串口连接器。

## 常见问题

### Q1: GPIO 14/15 串口不工作

- **原因**：固件禁用了 GPIO 14/15 的 UART 复用功能
- **解决**：使用板子上独立的专用串口连接器

### Q2: 串口无输出

- 检查 TX/RX 是否接反
- 检查波特率是否正确（115200）
- 检查串口模块供电是否正常
- 确认连接的是专用串口而非 GPIO

### Q3: 乱码

- 波特率不匹配
- 数据位/校验位/停止位设置错误
- 尝试 8N1（8数据位，无校验，1停止位）

### Q4: ESP8266 连接失败

- 确认模块已配置为透传模式
- 确认电脑和 ESP8266 在同一网络
- 检查 TCP 端口是否被占用
